<decisions_made>

- Engine timing: switched from `t0 + timestamp/speed + extra_delay` to gap-based virtual
  clock (`virtual_time += delta`). Handles manually inserted blocks (t=0), reordered
  blocks, and Record Here appended blocks (timestamps restart from 0) correctly.
- EditRole for DelayBlock returns bare "0.500" (no "s") so inline editor pre-fills with
  a number the user can directly overwrite. _set_block_value strips "s" suffix defensively.
- setData uses dataChanged not beginResetModel — display rows don't change structurally
  on inline value edits, so full reset was wrong and caused Qt commitData warning.
- BlockTypeDialog returns copy.copy(proto) so each inserted block is independent.
- "Record Here" button added to EditorPanel toolbar — emits record_here_requested(flat_index),
  MainWindow handles append-mode recording in _start_record_here().
