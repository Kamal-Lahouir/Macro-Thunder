  <files>src/macro_thunder/engine/__init__.py, tests/test_engine.py</files>
  <behavior>
    PlaybackEngine(mouse_ctrl=None, kb_ctrl=None, on_progress=None):
      - mouse_ctrl: pynput.mouse.Controller (or mock). If None, create real Controller.
      - kb_ctrl: pynput.keyboard.Controller (or mock). If None, create real Controller.
      - on_progress: callable(index: int, total: int) | None — called AFTER each block dispatch.
        Called from the playback thread — caller must use a queue to bridge to Qt.
      - _stop_event: threading.Event(), initially clear.

    start(blocks: list[ActionBlock], speed: float = 1.0, repeat: int = 1) -> None:
      - Clears _stop_event
      - Starts daemon threading.Thread targeting _run(blocks, speed, repeat)
      - Returns immediately (non-blocking)

    stop() -> None:
      - Sets _stop_event
      - Safe to call before start(), during playback, or after completion

    _run(blocks, speed, repeat):
      for _ in range(repeat):
          t0 = time.perf_counter()
          for i, block in enumerate(blocks):
              if _stop_event.is_set():
                  return
              target = t0 + block.timestamp / speed
              remaining = target - time.perf_counter()
              if remaining > 0.002:
                  time.sleep(remaining - 0.002)
              while time.perf_counter() < target:
                  pass
              _dispatch(block)
              if on_progress:
                  on_progress(i + 1, len(blocks))

    _dispatch(block):
      - MouseMoveBlock: _mouse_ctrl.position = (block.x, block.y)
      - MouseClickBlock: btn = mouse.Button[block.button]
                         if block.direction == "down": _mouse_ctrl.press(btn)
                         else: _mouse_ctrl.release(btn)
      - MouseScrollBlock: _mouse_ctrl.scroll(block.dx, block.dy)
      - KeyPressBlock: key = _parse_key(block.key)
                       if block.direction == "down": _kb_ctrl.press(key)
                       else: _kb_ctrl.release(key)
      - DelayBlock, LabelBlock, GotoBlock, WindowFocusBlock: no-op for now (Phase 4)

    _parse_key(key_str: str):
      - starts with "Key." → keyboard.Key[key_str[4:]]  (e.g. "Key.shift" → keyboard.Key.shift)
      - else → keyboard.KeyCode.from_char(key_str)

    Test cases (all use mock controllers, no OS hooks):
      DISPATCH tests (synchronous — call _dispatch() directly):
        - MouseMoveBlock(x=100, y=200, timestamp=0.0) → mock_mouse.position set to (100, 200)
        - MouseClickBlock(x=0, y=0, button="left", direction="down", timestamp=0.0) → mock_mouse.press(Button.left)
        - MouseClickBlock(x=0, y=0, button="right", direction="up", timestamp=0.0) → mock_mouse.release(Button.right)
        - MouseScrollBlock(x=0, y=0, dx=0, dy=-3, timestamp=0.0) → mock_mouse.scroll(0, -3)
        - KeyPressBlock(key="Key.shift", direction="down", timestamp=0.0) → mock_kb.press(Key.shift)
        - KeyPressBlock(key="a", direction="up", timestamp=0.0) → mock_kb.release(KeyCode.from_char('a'))

      STOP test:
        - Build 3 blocks with timestamp=0.0 (tight timing)
        - start(blocks, speed=1.0, repeat=100)
        - sleep 0.05s then call stop()
        - join thread (timeout=2s) → thread must have exited
        - Assert dispatched count < 300 (did not finish all repeats)

      PROGRESS test:
        - Build 2 blocks with timestamp=0.0
        - Collect progress tuples via on_progress callback
        - After completion: assert progress == [(1, 2), (2, 2)]

      SPEED test (structural, not timing):

        - With speed=2.0 and block.timestamp=1.0: target should be t0 + 0.5
        - Test by mocking time.perf_counter() if desired, or just verify dispatch order is correct at speed=10x

      REPEAT test:
        - 2 blocks, repeat=3 → on_progress called 6 times total
