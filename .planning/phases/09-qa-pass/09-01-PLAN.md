<objective>
Fix the two confirmed playback/recording bugs:

1. Stale sentinel wipes amber cursor — engine puts `(-1,-1)` into the progress queue on natural completion; if the user already pressed the stop hotkey, the drain loop still calls `_stop_play(clear_cursor=True)` and erases the amber position. Fix: guard the sentinel branch on `self._state == AppState.PLAYING`.

2. Stop-hotkey key-up leaks into recorded blocks — `_on_press` suppresses the stop key press but `_on_release` has no matching guard. Fix: add `_stop_key_consumed` bool flag to `RecorderService`.

Purpose: These are the two highest-priority bugs; they break the core record/play loop.
Output: Two targeted one-line fixes, two regression test files.
</objective>
