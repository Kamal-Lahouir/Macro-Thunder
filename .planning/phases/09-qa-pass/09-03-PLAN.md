  <how-to-verify>
    Launch the app: python -m macro_thunder

    **Test 1 — Amber cursor / resume (Success Criterion 1)**
    1. Load or record any macro with at least 10 blocks
    2. Press Play (toolbar or Play hotkey) — playback starts, amber row advances
    3. While playing, press the Stop hotkey (e.g. F10) — playback halts; amber row stays on the stopped position (does NOT disappear)
    4. Press Play hotkey again — playback resumes from the amber row, not from row 0
    PASS if: amber stays after stop, resumes from amber on next play

    **Test 2 — Recording residue (Success Criterion 2)**
    1. Press Record hotkey to start recording
    2. Press one or two keys (e.g. A, B)
    3. Press the Stop hotkey to stop recording
    4. Inspect the block list — there must be NO block for the stop key's key-up event
    PASS if: block list contains only the intentionally pressed keys; no extra KeyPress block for the stop key

    **Test 3 — Block edit dialog (Success Criterion 3)**
    For each block type below, double-click a row of that type and confirm the correct dialog opens:
    - MouseMoveBlock: dialog shows X, Y fields
    - MouseClickBlock (down): dialog shows X, Y, button, direction; find paired "up" block; edit X on "down" → "up" block X also changes
    - MouseScrollBlock: dialog shows X, Y, dx, dy fields
    - KeyPressBlock: dialog shows key capture field; click "Press a key...", press Escape — field shows "Key.esc"
    - DelayBlock: dialog shows duration float field
    - LabelBlock: dialog shows name field
    - GotoBlock: dialog shows target label field
    - WindowFocusBlock: dialog shows all fields (executable, title, match_mode, timeout, labels, reposition)
    - LoopStartBlock: dialog shows repeat count spinbox
    - LoopEndBlock: double-click does NOT open any dialog
    After confirming any edit (click OK), confirm the macro dirty indicator (title bar asterisk or "unsaved" label) appears.
    Click Cancel on another block — confirm its value is unchanged.
    PASS if: all dialogs open correctly, dirty flag set on OK, Cancel leaves unchanged, LoopEnd skips dialog

    **Test 4 — Flow control (Success Criterion 4)**
    1. Build a macro: LabelBlock name="top", DelayBlock 0.1s, GotoBlock target="top"
    2. Press Play — after ~1-2 seconds a warning dialog "Infinite Loop Detected" should appear
    PASS if: warning fires and playback stops (loop detection working)
    3. Build a macro: LoopStartBlock repeat=3, DelayBlock 0.05s, LoopEndBlock
    4. Press Play — should complete 3 iterations (total ~0.15s delay), then stop naturally
    PASS if: playback completes cleanly without hanging

    **Test 5 — Button state / toolbar blink (Success Criterion 5)**
    1. Press Play, then press Stop hotkey — toolbar Play button must NOT appear active after stop
    2. Start recording, stop recording — the blinking record indicator must disappear
    3. With no macro loaded, Play button should be disabled (or clicking it does nothing)
    PASS if: no mismatched button states; blink disappears on stop

    Type "approved" if all 5 tests pass, or describe which test(s) failed with exact symptoms.
  </how-to-verify>
