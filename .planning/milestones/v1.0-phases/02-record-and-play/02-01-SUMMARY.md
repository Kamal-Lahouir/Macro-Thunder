# Phase 02 Plan 01: RecorderService Summary

**One-liner:** pynput mouse+keyboard capture pipeline with pixel threshold filtering and injected-event suppression, fully testable without OS hooks.

## What Was Built

`RecorderService` is the input capture pipeline for Phase 2. It wraps two pynput `Listener` instances (mouse and keyboard) and converts raw OS callbacks into typed `ActionBlock` instances placed into a `queue.Queue`.

Key design properties:
- Zero Qt imports — safe to call from any thread
- Pixel threshold (default 3px Euclidean) discards micro-jitter mouse moves
- Win32 `LLMHF_INJECTED` / `LLKHF_INJECTED` bit filters suppress events generated by playback engine, preventing feedback loops
- `start()` / `stop()` lifecycle; `stop()` is idempotent and safe before `start()`
